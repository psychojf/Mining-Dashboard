## How to Use EVE Mining Dashboard


### Managing Characters & Config

Your characters are detected automatically from your EVE game logs.
Click the gear icon (⚙) in the top-right corner of the hub to open the Config dialog.

The Config dialog is a single scrollable panel with all settings visible at once:

  ◆ CHARACTER SELECTION
    - Shows all detected pilots in a 3-column grid.
    - Check or uncheck each pilot to show or hide them on the dashboard.
    - Use SELECT ALL / DESELECT ALL to quickly toggle all at once.
    - Up to 15 pilots are shown without scrolling; more than 15 activates a scrollbar.

  ◆ PATHS & FILES
    - Gamelogs Path : Change this if your EVE logs are in a non-default location.

  ◆ APPEARANCE & LIMITS
    - Dashboard Theme    : Choose from 20+ EVE faction colour themes. Selecting one applies
                           it immediately in-place — the Config dialog stays open and recolors
                           live without closing or flickering.
    - Default History Days : How many days back the History window analyzes by default.
    - Window Transparency  : Slider to set the opacity of all dashboard windows (20%–100%).
    - Play audio on Critical Hit : Toggle the audio alert on critical mining successes.

  ◆ FLEET
    - Webhook URL : Paste a Discord webhook URL to enable fleet reporting (see Fleet Mode below).

  ◆ ORE DATABASE (SDE)
    - Displays the current ore database version, ore count, and last update timestamp.
    - Click UPDATE ORE DATA to download the latest EVE SDE ore volumes and compression ratios.
      A progress bar appears during the download.

Click SAVE to apply all changes, or CANCEL to discard.


### ISK/Hour Display

Each character frame shows a live ISK/h value based on ESI market prices:

  - Displayed as "◈ Value: 229.23 M ISK/h" (millions, 2 decimal places).
  - Prices are fetched from the EVE ESI API on startup and cached locally.
  - Ore type IDs are resolved automatically via the ESI universe/ids endpoint.
  - If the value shows "--", a hint explains why (fetching prices, no ore yet, resolving IDs, etc.).


### Auto-Stop Toggle (per character)

By default, the session pauses automatically when your cargo hold is full.
Each character frame has an ⏸ Auto-Stop button in the cargo section:

  - ⏸ Auto-Stop: ON  (cyan)  — session pauses when cargo is full (default behaviour).
  - ⏸ Auto-Stop: OFF (dim)   — cargo-full events are ignored; session keeps running.

Click the button to toggle. The setting is saved per character and persists across restarts.
Note: other auto-pause triggers (asteroid depleted, targeting failed) are always active.


### Main Window Controls

The main dashboard is a floating control hub. The title bar contains:

  - Pin (📌)         — Toggles always-on-top for the hub. Lit (cyan) = pinned.
  - Config (⚙)       — Opens the Config dialog.
  - Minimize (–)     — Sends the dashboard to the system tray.
  - Close (✕)        — Exits the application completely.
  - Resize grip (◢)  — Drag the bottom-right corner to resize the hub window.
  - Double-click title bar — Rolls the window up to just the title bar (window shade).
                             Double-click again to restore.


### System Tray

When minimized, the dashboard lives in your system tray (bottom-right taskbar).

  - Left-click  the tray icon → restore the dashboard.
  - Right-click the tray icon → Show Dashboard  /  EXIT.


### Character Fleet Hub

Each character enabled in Config gets their own floating window when the dashboard launches.
The hub displays your ACTIVE MINING FLEET roster:

  - ◉ HIDE — Temporarily hides a character's floating window without removing them from the fleet.
             Useful for clearing screen space while still tracking them.
  - ◉ SHOW — Brings back a previously hidden character's floating window.

To permanently remove a character from the fleet, uncheck them in ⚙ Config.


### Character Frames (Floating Windows)

Each character has an independent always-on-top floating window.

  - Move        : Click and drag the title bar (e.g. "★ JF GELINAS") to reposition.
  - Resize      : Drag the small resize grip (◢) in the bottom-right corner.
  - Auto-Fit    : Double-click the title bar to snap back to the default fitted size.
  - Hide        : Click ✕ on the title bar to hide the window (stays in the fleet).
                  Use ◉ SHOW in the hub to bring it back.
  - Session Breakdown : Click "v SESSION BREAKDOWN v" at the bottom to expand/collapse
                        the per-ore-type stats for the current session.
  - Persistence : Window sizes and positions are saved automatically and restored on next launch.


### Configuring Ship Fittings

Right-click anywhere on a character's floating window and select ⚙ Ship Config.

  Profiles
    Create multiple ship profiles (e.g. "Ore Barge", "Ice Miner", "Gas Huffer") using the
    + NEW, ✎ RENAME, and ✕ DELETE buttons. Switch between them via the dropdown, or click
    the gold profile name directly on the character frame.

  Modules
    Configure up to 5 mining modules per profile. For each:
      - Module Name      : A label for your reference (e.g. "Strip Miner I")
      - Yield (m3/cycle) : How many m3 the module extracts per cycle (check your fitting window in EVE)
      - Cycle Time (s)   : Duration of one cycle in seconds
    Use the checkboxes to enable/disable individual modules. The Theoretical rate preview updates live.

  Drones
    Configure active mining drones — their individual yield and cycle time.

  Implant
    Toggle the Highwall MX-1005 implant for a +5% module yield boost.

  Cargo Hold
    Set your ship's cargo capacity to enable the cargo progress bar and time-to-full estimates.
    When you compress ore in-space, the bar automatically adjusts to reflect the reduced volume.
    Multi-word ore names (e.g. "Compressed Arkonor") are fully supported.

  Click SAVE to apply the fitting.


### Starting a Session

  1. Set up your ship fitting and position the character window where you want it on screen.
  2. Click ▶ START on a character's frame to begin tracking.
  3. Mine as usual in EVE — stats, critical hits, cargo bar, ISK/h, and actual m3/s update in real time.
  4. The Residue stat tracks ore volume lost to compression residue (normal with T2 crystals).
  5. Click "v SESSION BREAKDOWN v" to see the per-ore detail.
  6. Click ■ STOP to pause tracking without clearing data.
  7. Click ↺ RESET to clear all session stats and cargo for that character.
  8. Click ⏏ EMPTY to reset only the cargo bar while keeping all session stats intact.

  Auto-Pause:
    The session pauses automatically (returns to ▶ START) when EVE logs any of:
      - Your cargo hold is full  (can be disabled per character with ⏸ Auto-Stop: OFF)
      - The asteroid you are mining is depleted
      - A targeting attempt failed (object no longer present)
    Simply click ▶ START again to resume after repositioning or switching rocks.


### Compression & Cargo Tracking

When you compress ore in-space, the dashboard detects the compression event from your logs and
automatically adjusts the cargo bar:
  - The raw (uncompressed) volume is removed.
  - The compressed volume is added back (raw / compression ratio).
  - Net result: the bar drops significantly but does NOT go to zero — it reflects what is actually
    in your hold after compression.
  - Works for all ore names including multi-word types (e.g. "Compressed Crimson Arkonor").

Note on Residue: If you use Modulated Strip Miner II with T2 crystals, EVE's Scrapmetal Processing
skill causes genuine ore loss logged as "Additional X units depleted from asteroid as residue."
This is normal EVE game behaviour, not a dashboard bug. The Residue stat reflects this accurately.


### Fleet Mode & Discord Reporting

  1. Open ⚙ Config → FLEET and paste your Discord webhook URL.
  2. Once a session has active data, two buttons appear on the character frame:
       - ⎘ Copy to Clipboard : Copies a plain-text report (duration, crits, total m3, ore breakdown,
                               ISK/h value).
       - ▲ Send to Discord   : Opens a preview, then posts the report to your Discord server.


### Viewing Mining History

Click the ◈ HISTORY button at the bottom of the main hub.

  - Enter the number of days to analyze and press Enter or click ↻ REFRESH STATS.
  - The history window shows total m3 mined per character with a per-ore breakdown from your saved logs.
  - ◈ EXPORT EXCEL : Generates a .xlsx file with a Summary sheet, Ore Pivot tables, and daily
                     breakdowns for each character.


### Configuration File

All settings are saved automatically to mining_config.json in the same folder as the script:

  - Window positions and sizes (hub, history, config, and each character frame)
  - Character visibility selections and hidden states
  - All ship profiles, modules, drones, implant, and cargo configurations
  - Application settings (log path, webhook, theme, sound toggle, transparency)
  - Auto-Stop preference per character

Note: Delete mining_config.json to completely reset the application to its default state.
